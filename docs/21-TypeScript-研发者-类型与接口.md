# TypeScript 研发者：类型与接口

## 类型与接口

- 对导出的函数、共享工具和公共方法显式标注参数类型与返回类型
- 局部变量如果类型明显，可交给 TypeScript 推断
- 重复出现的内联对象结构应提取为具名 `type` 或 `interface`

## `interface` 与 `type`

- 对可扩展、可实现的对象结构优先使用 `interface`
- 对联合类型、交叉类型、元组、映射类型和工具类型优先使用 `type`
- 除非出于兼容性需要，否则优先使用字符串字面量联合，而不是 `enum`

## 避免 `any`

- 应用代码中避免使用 `any`
- 对外部或不可信输入优先使用 `unknown`，再安全收窄
- 当返回值依赖调用方时，优先使用泛型

## React 属性

- 组件属性应使用具名 `interface` 或 `type`
- 回调属性要显式标注类型
- 除非有明确理由，否则不要默认使用 `React.FC`

## JavaScript 文件

- 在 `.js` 和 `.jsx` 中，如果迁移到 TypeScript 不现实，但类型有助于理解，应使用 JSDoc
- JSDoc 的定义应与运行时行为保持一致
