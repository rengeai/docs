# TypeScript 研发者：不可变与边界处理

## 不可变写法

- 始终优先返回新对象，不要原地修改已有对象
- 更新对象或数组时，优先使用展开运算符、`map`、`filter` 等不可变写法

这样做的原因是：

- 降低隐藏副作用
- 让调试更直接
- 使并发执行更安全

## 错误处理

- 优先使用 `async/await` 配合 `try/catch`
- 对 `unknown` 错误做安全收窄
- 在界面层提供用户可理解的错误信息
- 在服务端记录足够详细的错误上下文
- 不要静默吞掉错误

## 输入校验

- 所有系统边界输入都必须校验
- 优先使用基于模式的校验工具，例如 `Zod`
- 对用户输入、接口返回、文件内容等外部数据都保持不信任
- 出错时应尽早失败，并返回清晰信息

## 日志

- 生产代码中不要保留 `console.log`
- 使用正式日志方案替代调试输出
