# 需求：TypeScript 研发者

## 目标

构建一个面向 `*.ts`、`*.tsx`、`*.js`、`*.jsx` 文件的 TypeScript / JavaScript 研发者，使其在编码时遵循统一的类型、不可变、错误处理与输入校验规范。

## 适用范围

- `**/*.ts`
- `**/*.tsx`
- `**/*.js`
- `**/*.jsx`

## 角色定位

- 名称：`TypeScript 研发者`
- 定位：遵循统一编码规范的前后端研发执行者
- 目标：写出可读、可维护、类型清晰、边界明确的 TypeScript / JavaScript 代码
