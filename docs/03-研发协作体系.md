# 研发协作体系

## 总体原则

研发协作体系参考 GitLab 的 Issue、路线图和 PR 模型，但要为 AI 智能体提供更强的结构化分工和可追踪性。

## Issue 体系

Issue 是一切工作的起点，也是需求、缺陷、任务和讨论的统一容器。

### Issue 的作用

- 承载需求描述
- 拆解执行任务
- 记录讨论过程
- 追踪状态变化
- 连接目标、路线图和交付结果

### 建议的 Issue 类型

- Epic 级议题：跨周期、跨项目的重要主题
- Feature 级议题：具体功能建设
- Task 级议题：明确可执行的工作项
- Bug 级议题：缺陷修复
- Research 级议题：方案调研与验证
- Ops 级议题：运维、发布和支持事项

### 建议的 Issue 字段

- 标题
- 背景
- 目标
- 验收标准
- 优先级
- 负责人
- 协作智能体
- 关联项目
- 关联 OKR
- 状态

## 路线图体系

路线图用于承接中长期规划，是 Issue 集合在时间维度上的表达方式。

### 路线图需要回答的问题

- 当前阶段最重要的建设方向是什么
- 每个方向由哪些项目承接
- 每个项目由哪些 Epic 或 Feature 组成
- 当前进度如何
- 风险点在哪里

### 建议的路线图层次

- 战略路线图：对应组织级目标
- 项目路线图：对应单个项目的季度或月度安排
- 执行路线图：对应当前迭代内的重点工作

## PR 体系

PR 是变更进入主线的治理入口，也是人与 AI 协作质量控制的关键环节。

### PR 的目标

- 解释为什么要修改
- 描述修改了什么
- 展示如何验证
- 暴露潜在风险
- 沉淀评审过程

### AI 场景下的 PR 特点

- PR 可以由 AI 智能体发起
- PR 必须附带结构化变更说明
- 关键 PR 需要人类或高权限智能体审批
- 审批意见和测试结果要形成可追踪记录

### 建议的 PR 模板字段

- 背景
- 变更范围
- 验证方式
- 风险说明
- 回滚方案
- 关联 Issue
- 关联目标

## 从 Issue 到 PR 的闭环

建议形成如下闭环：

1. 从目标或路线图产生 Issue
2. 由人类或 AI 智能体认领 Issue
3. 在项目内推进设计与实现
4. 通过 PR 进入评审流程
5. 合并后更新路线图和目标状态
6. 在复盘中沉淀经验与制度优化

这个闭环是未来整个系统稳定运行的基础。
