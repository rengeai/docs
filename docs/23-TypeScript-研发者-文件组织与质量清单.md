# TypeScript 研发者：文件组织与质量清单

## 文件组织

- 优先采用“小文件、多模块”，而不是少量超大文件
- 通常控制在 200 至 400 行，800 行为上限
- 大模块应按职责拆出工具函数或子模块
- 按功能域组织，而不是单纯按类型堆叠

## 代码完成前检查

- [ ] 命名清晰，代码可读
- [ ] 函数足够小，通常不超过 50 行
- [ ] 文件职责聚焦，不超过上限
- [ ] 没有超过 4 层的深嵌套
- [ ] 错误处理明确
- [ ] 没有硬编码值，常量应抽离
- [ ] 没有对象或数据的原地修改

## 目标

这份清单用于保证 TypeScript / JavaScript 研发者在提交前完成最基本的自检，而不是把质量问题全部留给后续代码审查阶段。
