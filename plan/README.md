# Plan 提案索引

本目录采用 BIPs 风格的“提案仓库机制”管理需求计划。

- 提案文件命名为 `PLAN-<IssueID>-xxxxx.md`
- 每份提案必须包含统一元数据头
- `Plan-ID` 是唯一追踪键，取自 Issue ID

提交与审阅规范请看 [CONTRIBUTING](./CONTRIBUTING.md)。  
提案模板与状态流请看 [PLAN-0001-提案流程与模板](./PLAN-0001-提案流程与模板.md)。

## 目录结构

- `README.md`：提案总表与状态看板
- `CONTRIBUTING.md`：提交流程、编号规则、门禁说明
- `PLAN-0001-提案流程与模板.md`：提案规范
- `assets/`：公共图片、流程图资源
- `PLAN-<IssueID>-xxxxx.md`：具体提案正文
