📄 PRD — GitLab × OpenClaw Problem Definition System（MVP）

⸻

1. Introduction / Overview

本产品是一套基于 GitLab 的 AI 协作系统，面向技术型产品团队，在产品设计阶段引入多 Agent 协作机制。

系统以 GitLab 为唯一事实来源（Single Source of Truth），通过 Issue 驱动流程，利用 OpenClaw Agent 参与问题定义过程，将非结构化输入转化为结构化问题定义文档。

本阶段仅聚焦一个核心能力：

将“会议后的混乱信息”转化为“结构清晰的问题定义文档”

⸻

2. Goals / Objectives

核心目标（MVP）
	•	在 1 次流程内，将非结构化输入转化为完整问题定义文档
	•	问题定义文档结构完整率 ≥ 100%
	•	问题定义中“未解决风险项”数量 = 0
	•	全流程在 GitLab Issue 内完成（无外部依赖）

辅助目标
	•	减少产品经理手动整理问题时间 ≥ 70%
	•	提高问题定义一致性（不同人输出差异降低）

⸻

3. Target Audience / User Personas

🎯 主要用户：技术型产品团队

产品经理（Primary User）
	•	输入会议记录
	•	获取结构化问题定义
	•	推动后续产品流程

仲裁者（Secondary User）
	•	在问题无法收敛时介入
	•	不参与常规流程

⸻

4. User Stories

US-1（核心主线）

作为产品经理，在开会后面对混乱的讨论记录，我希望系统自动将其转化为结构化问题定义文档，以便进入下一步产品设计。

⸻

US-3（质量控制）

作为产品负责人，我希望系统自动识别问题定义中的缺失、矛盾或不确定点，并推动修正，以确保输出质量。

⸻

US-5（收敛机制）

作为 Problem Owner Agent，我需要整合多方输出，生成唯一且最终的问题定义版本。

⸻

5. Functional Requirements

⸻

5.1 Issue 输入模块

描述：
用户创建 GitLab Issue，输入原始需求内容（会议记录等）

要求：
	•	支持非结构化文本输入
	•	不强制格式
	•	支持基础标签（如 problem-definition）

⸻

5.2 Agent 触发机制

触发条件：
	•	Issue 状态 = NEW
	•	Issue 被指派给 Agent

行为：
	•	自动触发 Problem Agent 执行

⸻

5.3 问题结构化生成模块

执行者： Problem Agent

输出必须符合模板：
	1.	问题标题
	2.	背景（仅客观事实）
	3.	核心问题
	4.	目标
	5.	范围
	6.	非范围
	7.	约束条件
	8.	关键风险/歧义
	9.	验收口径
	10.	待确认问题

约束：
	•	禁止模糊表达（如：可能、大概）
	•	必须字段完整

⸻

5.4 自动校验模块

执行者： Validator Agent / 系统规则

检查内容：
	•	字段是否完整
	•	是否存在逻辑矛盾
	•	是否存在不确定表达
	•	是否存在未解决问题

结果：
	•	若存在问题 → 标记 RISK
	•	触发修正流程

⸻

5.5 Problem Owner 收敛模块（核心）

执行者： Problem Owner Agent

职责：
	•	汇总多 Agent 输出
	•	输出唯一最终版本

权限：
	•	标记最终版本
	•	推动 Issue 状态变更

⸻

5.6 完成判定机制

必须满足以下条件：
	•	所有字段完整
	•	无 RISK 标记
	•	存在 Problem Owner 最终版本

结果：
	•	Issue 状态从 NEW → DEFINED

⸻

5.7 仲裁机制（弱化版）

触发条件：
	•	多轮修正仍存在 RISK
	•	Agent 输出存在冲突未解决

行为：
	•	通知仲裁者介入
	•	仲裁者可直接修改或指导 Agent

⸻

6. Non-Functional Requirements

性能
	•	单个 Issue 完整处理时间 ≤ 2分钟（MVP目标）

可控性
	•	所有操作必须可追溯（基于 GitLab）

稳定性
	•	输出结构必须一致（模板强约束）

安全性
	•	不允许访问 GitLab 之外数据
	•	不允许 Agent 之间私下通信

⸻

7. Design Considerations
	•	UI 基于 GitLab 原生界面（不做额外前端）
	•	所有交互通过：
	•	Issue Comment
	•	状态流转
	•	输出以 Markdown 结构呈现

⸻

8. Success Metrics
	•	问题定义完成率 ≥ 95%
	•	RISK 标记清除率 = 100%
	•	平均处理轮次 ≤ 3轮
	•	用户手动修改比例 ≤ 20%

⸻

9. Open Questions / Future Considerations
	•	是否引入 PRD Agent（下一阶段）
	•	是否支持多 Issue 关联（复杂需求）
	•	是否引入协作追溯（User Story 2）
	•	是否增强冲突检测机制
