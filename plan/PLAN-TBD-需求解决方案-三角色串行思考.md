# 需求解决方案：三角色串行思考

## 背景

当前需求评审常出现“跳步、脑补、结论不清”的问题，导致方案质量不稳定。

## 问题定义

需要一套可重复执行的 AI 思考链路，把“问题定义、产品方案、价值裁决”拆成严格串行的三段。

## 范围与非范围

- 范围：Problem Agent、Product Agent、ROI Agent 三角色串行协作。
- 非范围：并行协作、多角色混合输出、自由补充上下文。

## 方案设计

执行顺序固定为：

`Problem Agent -> Product Agent -> ROI Agent`

每个 Agent 只允许使用上一个 Agent 的结构化输出，不允许使用未声明信息，不允许脑补。

每个 Agent 必须同时完成三项动作：

1. 挑错：至少提出 3 个明确问题或风险点
2. 收敛：输出唯一结论与执行建议
3. 明确表达：禁止模糊词（例如：可能、大概、可以考虑）

## 执行计划

1. 输入原始需求到 Problem Agent。
2. Problem Agent 输出“问题定义稿（含 3 个以上挑错 + 收敛结论）”。
3. Product Agent 仅基于上一步输出，产出“产品方案稿（含 3 个以上挑错 + 收敛结论）”。
4. ROI Agent 仅基于上一步输出，产出“价值判断与决策（含 3 个以上挑错 + 最终裁决）”。
5. 将 ROI 结果作为提案结论写回 Issue 和 MR。

## 风险与回滚

- 风险：角色跳步导致结论不可追溯。
- 风险：使用模糊语言导致执行歧义。
- 回滚：若任一阶段不满足三项动作，退回上一步重做。

## 验收标准

- 三个 Agent 输出均含“挑错 >= 3 + 收敛结论 + 无模糊词”。
- 全链路为严格串行，后一步仅引用前一步输出。
- 最终结论可直接用于立项或终止决策。
