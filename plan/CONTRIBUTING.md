# Plan 贡献指南

## 提交流程

1. 先在 `docs` 仓库创建 Issue（会话、用户故事、用户旅程、会议记录均可）。
2. 在 Issue 讨论收敛后，创建提案文件并发起 MR。
3. 提案进入 `Review`，等待审阅与裁决。
4. 董事会通过后状态改为 `Approved`，并允许进入研发执行链。

## 编号规则

- 作者不得自行分配 `Plan-ID`。
- `Plan-ID` 由维护者在合并前分配，格式为 `PLAN-XXXX`。
- 文件名可为中文标题，但必须在元数据中记录 `Plan-ID`。

## 审阅门禁

- 未 `Approved` 的提案不得创建研发里程碑和研发任务。
- 项目经理代理必须“人工确认后创建”研发执行对象。
- 研发任务和 MR 必须继承追溯字段：
  - `docs_issue_id`
  - `plan_doc_path`
  - `plan_commit_sha`

## 最低质量要求

- 元数据头字段完整且可追溯。
- 问题定义、范围边界、验收标准明确。
- 与现有提案无明显冲突和重复。
