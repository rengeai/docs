# RengeAI 文档库

这是一个面向中文场景的 Markdown 文档库，用于系统化描述 RengeAI 接下来要构建的事情、背后的方法论，以及一套基于 GitLab 的多 OpenClaw 智能体协作模式。

## 项目定位

RengeAI 不只是一个单体产品，而是一套围绕多 OpenClaw 智能体协作展开、并建立在 GitLab 语义之上的组织化工作系统。

从产品形态上看，它并不是要替代 GitLab 或 OpenClaw，而是要在两者之上构建一层更易搭建、更易治理、更易长期维护的统一系统界面与制度框架。

它希望解决的问题是：

- 如何让多个 AI 智能体像一个真正的团队一样协同工作
- 如何让团队在多组织、多项目的环境下保持清晰的权限边界
- 如何让需求、研发、交付和复盘形成统一的治理框架
- 如何让项目推进既有工程秩序，也有战略对齐

## 核心设计方向

- 组织结构、项目层级和权限模式，全面参考 GitLab 的体系
- 项目管理的方法与理念采用 PMP，作为治理与推进的基础框架
- 协作流程包括 GitLab 风格的 Issue、里程碑、MR 等核心机制
- 目标管理采用 OKR 体系进行目标拆解和推进
- 日常协作与交付主要使用 GitLab 作为工具承载
- 产品设计从 MVP 出发，优先统一接入、治理和最小协作闭环
- 所有制度设计最终都要服务于 AI 智能体团队的规模化协作

## 文档原则

- 中文优先：所有核心文档优先使用中文编写
- 结构优先：优先把框架搭清楚，再逐步填充细节
- 可执行优先：文档不仅用于表达理念，也要能指导后续实施
- 长期演进：所有文档都应当支持版本化迭代
