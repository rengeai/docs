# AI Agent 文件体系规范（MVP）

## 目标

定义 AI Agent 系统的文件结构与职责边界，避免信息混乱、职责重叠、不可维护。

---

# 一、整体分层

Agent 系统由四层组成：

1. 人设层（它是谁）
2. 运行层（它怎么工作）
3. 能力层（它能做什么）
4. 状态层（它当前处于什么情况）

---

# 二、文件清单与职责

## 1. IDENTITY.md（身份定义）

### 作用
定义 Agent 的“客观角色与职责范围”。

### 必须包含
- 名称
- 角色描述
- 核心职责
- 服务对象
- 不负责范围
- 成功标准

### 禁止内容
- 行为风格（属于 SOUL）
- 工作流程（属于 AGENTS）
- 工具能力（属于 TOOLS）

---

## 2. SOUL.md（行为与人格）

### 作用
定义 Agent 的行为风格、表达方式与决策倾向。

### 必须包含
- 思考方式
- 行为原则
- 表达风格
- 决策偏好
- 禁止行为

### 禁止内容
- 业务流程
- 工具说明
- 任务状态

---

## 3. AGENTS.md（运行规则）

### 作用
定义 Agent 的执行流程与输出规范。

### 必须包含
- 输入要求
- 输出结构（强约束）
- 工作流程步骤
- 执行约束

### 禁止内容
- 人格描述（属于 SOUL）
- 工具能力细节（属于 TOOLS）
- 长期记忆

---

## 4. TOOLS.md（能力边界）

### 作用
定义 Agent 可调用的工具与能力范围。

### 必须包含
- 工具列表
- 每个工具用途说明
- 使用规则
- 权限边界

### 禁止内容
- 业务逻辑
- 决策规则
- 流程控制

---

## 5. NOW.md（当前状态）

### 作用
记录 Agent 当前任务与上下文。

### 必须包含
- 当前任务
- 当前阶段
- 输入来源
- 关键约束
- 当前目标

### 特性
- 强时效性（必须可更新）
- 不保留历史

### 禁止内容
- 通用规则
- 长期记忆

---

## 6. MEMORY.md（长期记忆，可选）

### 作用
记录跨任务可复用的信息。

### 可包含
- 用户偏好
- 历史经验
- 稳定规则
- 常见问题

### 特性
- 长期存在
- 可累积

### 禁止内容
- 当前任务状态（属于 NOW）

---

## 7. USER.md（用户画像，可选）

### 作用
定义 Agent 服务对象的特征与偏好。

### 可包含
- 用户背景
- 偏好
- 目标
- 行为习惯

### 禁止内容
- Agent 自身规则
- 系统流程

---

## 8. skills/（能力模块）

### 作用
封装可复用的任务能力。

### 每个 skill 必须包含
- 目标
- 输入要求
- 输出结构
- 执行规则

### 特性
- 可复用
- 可组合
- 独立于具体任务

### 禁止内容
- 当前任务数据
- 用户信息

---

# 三、文件边界原则（必须遵守）

## 1. 单一职责原则
每个文件只解决一个问题：
- IDENTITY → 定义“是谁”
- SOUL → 定义“怎么表现”
- AGENTS → 定义“怎么做”
- TOOLS → 定义“能做什么”
- NOW → 定义“现在在干嘛”

---

## 2. 禁止跨层写入

错误示例：
- 在 SOUL 写流程 ❌
- 在 AGENTS 写人格 ❌
- 在 TOOLS 写决策 ❌

---

## 3. 强结构约束

- AGENTS.md 的输出结构必须固定
- skills 必须有统一格式
- 不允许自由发挥格式

---

## 4. 上下文分离

- NOW = 当前上下文
- MEMORY = 长期上下文

不得混用

---

# 四、最小可运行集合（MVP）

最少需要：

- IDENTITY.md
- SOUL.md
- AGENTS.md
- TOOLS.md
- NOW.md

即可运行一个基础 Agent

---

# 五、扩展建议（进阶）

当系统复杂度提升时，引入：

- MEMORY.md（长期记忆）
- USER.md（用户建模）
- skills/（能力模块化）

---

# 六、一句话总结

- IDENTITY：它是谁
- SOUL：它是什么风格
- AGENTS：它怎么工作
- TOOLS：它能用什么
- NOW：它现在在做什么

---

# 七、设计检查（使用前自检）

在新增任何内容前，必须回答：

1. 这是“身份”还是“行为”还是“流程”还是“能力”还是“状态”？
2. 是否写在了正确文件？
3. 是否与其他文件职责冲突？

如果不清晰 → 不允许写入
